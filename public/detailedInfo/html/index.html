What is HTML? Browser Rendering

How Browsers Render HTML and CSS

📜 A Quick Note

When you open an HTML page in a browser (like Chrome, Firefox, or Edge), the browser acts like a builder 🏗️ that takes the HTML code and turns it into a real webpage you can see and interact with.


Step-by-Step: How Browsers Render HTML

1️⃣ Browser Reads the HTML
  • 👉 The browser starts at the top of your HTML file and reads it line by line, just like reading a book.
  • 👉 It follows all the instructions written inside the HTML tags.

2️⃣ Builds the "DOM" (Document Object Model) 🏗️
  • 👉 The browser turns the HTML code into a structure called the DOM (Document Object Model).
  • 👉 Think of the DOM as a blueprint or a family tree 🌳 of your webpage, showing how everything is connected.

Example:

  • 👉 The browser sees that <h1> is inside <body>, so it places Hello, world! at the top.
  • 👉 Then, it places the <p> tag right below it with the text inside.

3️⃣ Applies CSS (Styles the Page) 🎨
  • 🤔 If your page has CSS (colors, fonts, layouts), the browser applies those styles on top of the DOM.

Example

  • ✅ Now, the h1 text becomes blue! 🟦

4️⃣ Handles JavaScript (Makes it Interactive) ⚙️
  • 👲 If your page has JavaScript (buttons, animations, pop-ups), the browser runs the JavaScript to make things interactive.

Example

  • When you click the button, JavaScript runs and pops up a message saying "Hello!"

5️⃣ Paints Everything on the Screen 🎥
  • ✅ After processing the HTML, CSS, and JavaScript, the browser paints the final webpage onto your screen so you can see it!

Conclusion
  • 📖 Reads the HTML file.
  • 🏗️ Creates the DOM structure.
  • 🎨 Adds CSS to make it look nice.
  • ⚙️ Runs JavaScript for interactivity.
  • 🎥 Displays the final webpage on your screen!
And that’s how your browser renders a webpage! 🚀

Character Encoding

📜 Quick Note

Character encoding tells the browser how to read and display text on a webpage, including special characters like ©, ü, or emojis 😊.


How Character Encoding Works? 🏗️

When you type Hello! 😊 in your HTML file, your computer doesn’t store letters or emojis. Instead, it stores numbers (called binary: 0s and 1s).

Example

  • H = 72
  • e = 101
  • 😊 = 128522

👉 Now, the browser needs to know how to turn these numbers back into readable text!

🥳 That’s where character encoding comes in! 🎉


Common Types of Character Encoding

1️⃣ UTF-8 (The Superhero! 🦸)
  • ✅ Can display ALL languages, symbols, and emojis! 🌍
  • ✅ This is why it’s the best choice for websites today!

Example in HTML

Why use it? It makes sure your page shows the right text, whether it's English, Arabic, Japanese, or even 🎙️🎨🚀!

2️⃣ ISO-8859-1 (The Old One 👴)
  • ❌ Only supports Western European languages (like English, French, Spanish).
  • ❌ Can’t handle special symbols or emojis properly.
  • 👎 Not recommended anymore.

3️⃣ UTF-16 (The Heavyweight 🏋️)
  • 💪 Stores characters in bigger chunks (uses more space).
  • 🏋️ Mostly used in special cases (like certain programming languages).

What Happens if You Use the Wrong Encoding? ❌

🤔 Imagine you open a book, but all the letters look broken! 📖💔

Example:

  • Instead of Café, you see CafÃ© – oh no! 😱

    • 👉 That happens when the browser doesn’t know the right encoding to use!


    Final Lesson: Always Use UTF-8! ✅

    And now, your web page can speak every language in the world! 🌎🚀
    Pretty cool, right? 😃

    HTML Entities & Special Characters

    📜 Quick Note

    Imagine you are writing a secret code 🕵️‍♂️, but some symbols are not allowed in normal text. HTML needs special codes to show certain characters correctly!

    These special codes are called HTML Entities! 🎉


    1️⃣ What Are HTML Entities? 🤔

    HTML doesn't understand some characters directly. For example:

  • If you write <, the browser thinks you’re starting a tag!
  • If you write ", the browser might get confused with an attribute!
    • 🔹 To fix this, we use HTML Entities (special codes that start with & and end with ;)

    2️⃣ Common HTML Entities 📜

    📌 Less Than (<) & Greater Than (>)
  • 👉 Used in HTML tags, but we need a special code to show them in text!

    • ✅ Output:

    <h1>Hello</h1>


    📌 Ampersand (&)

  • 👉 Used in links and code, but HTML mistakes it for an entity if not escaped!

    • ✅ Output: &


    📌 Double Quotes (") & Single Quotes (')

  • 👉 Used inside HTML attributes, but sometimes need to be escaped!

    • ✅ Output: "Hello" 'World'


    📌 Non-Breaking Space ( )

  • 👉 Creates extra spaces (normal spaces collapse in HTML)

    • ✅ Output: Hello World!


    3️⃣ Special Characters & Symbols 🎭

    📌 Copyright, Trademark & Registered Symbols

    ✅ Output:

    © ™ ®


    📌 Math Symbols ➕➖✖️

    ✅ Output:

    + − × ÷


    📌 Arrows & Pointers ➡️⬆️⬇️⬅️

    ✅ Output:

    → ← ↑ ↓


    📌 Currency Symbols 💰

    ✅ Output:

    € £ ¥ $


    4️⃣ When to Use HTML Entities? 🧐

    • ✅ When you need special characters that HTML might confuse (like <, >, &)
    • ✅ When writing non-breaking spaces ( ) to stop spaces from disappearing
    • ✅ When showing symbols (©, ™, €, →) without using Unicode

    Now go add cool symbols & escape characters in your HTML! 🚀🔥
